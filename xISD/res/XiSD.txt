┌─[О программе]───────────────────────────────────────────────────────────────
│    XiSD  -  это  плагин  к  FAR'у, который обеспечивает полноценную работу с
│ образами  дисков  системы  iS-DOS. Поддержка форматов образов осуществляется
│ через  плагины  второго  уровня, что  позволяет  добавлять  поддержку  новых
│ форматов без изменения базового плагина.
│    На настоящий момент поддерживаются 5 форматов образов: img, fdi, fdd, udi
│ и TeleDisk (только для чтения).
│    Текущая  версия  позволяет  выполнять  все  основные  файловые  операции:
│ копирование,  перемещение,  создание,  удаление  файлов  и  каталогов. Также
│ работают все  прелести  FAR'а,  например  поиск  файлов,  подсветка,  группы
│ сортировки.
│    Также  доступны  две специфические функции. Переименование файла/каталога
│ Shift+F6 и изменение атрибутов файла/каталога Ctrl+A.
│    Запрещено  переименовывать, изменять или удалять системные файлы, а также
│ изменять их атрибуты.
·
·
├─[Установка]─────────────────────────────────────────────────────────────────
│ 1. Создайте каталог ZX в каталоге FAR/Plugins
│ 2. Скопируйте все файлы и каталог Formats из архива в созданный каталог
│ 3. Перезагрузите FAR
·
·
├─[Конфигурация плагина]──────────────────────────────────────────────────────
│    В  диалоге  конфигурации  плагина Вы можете задать номер режима панели, в
│ котором  FAR  будет  показывать  iS-DOSные диски по умолчанию. Кроме того Вы
│ можете  настроить  4ый режим так как Вам удобно. Более подробно о настройках
│ режимов  отображения  читайте  в  help'е  FAR'а, следует отметить что плагин
│ передает  FAR'у  имя  файла,  размер,  дату  модификации,  атрибуты,  первый
│ блок  и  адрес  загрузки.  Последние  3  параметра  передаются  в колонках с
│ типами C0, C1 и C2 соответственно.
·
·
├─[Атрибуты файлов]───────────────────────────────────────────────────────────
│    Поскольку  атрибуты файлов и каталогов в системе iS-DOS отличаются от тех
│ которые поддерживает MS-DOS, было принято решение, что при копировании файла
│ на MS-DOSный диск атрибуты устанавливаются следующим образом:
│ ∙ файлы защищенные от чего либо получают атрибут ReadOnly
│ ∙ невидимые файлы получают атрибут Hidden
│    Буквы в атрибутах файла на панели плагина означают следующее:
│ ∙ S - сегментированный файл/каталог
│ ∙ R - файл/каталог защищен от чтения
│ ∙ W - файл/каталог защищен от записи
│ ∙ D - файл/каталог защищен от удаления
·
·
├─[Некоторые особенности]─────────────────────────────────────────────────────
│ ∙ Файлы _ВСЕГДА_ создаются непрерывные
│ ∙ Каталоги _ВСЕГДА_ создаются сегментированные размером в один блок
│ ∙ При копировании файлов теряются атрибуты
│ ∙ Системные файлы удалить нельзя в принципе
│ ∙ Адрес загрузки хитрым образом передается в виде даты создания файла,
│   посему если для конкретного файла он важен, то лучше дату создания не
│   трогать
│ ∙ Достаточно простая обработка недопустимых и длинных имен
│ ∙ Плагин  второго уровня для работы с форматом td0, входящий в состав XiSD,
│ имеет  несколько  ограничений
│   Во-первых, он умеет только читать файлы
│   Во-вторых, возможны ошибки при работе с повреждёнными файлами
│   В-третьих, поддерживается только новый (v2.xx) алгоритм компрессии
│ Если  вдруг  кто-нибудь  имеет  более  полную информацию об этом формате, не
│ сочтите за труд, поделитесь!
·
·
├─[Некоторые неприятные особенности]──────────────────────────────────────────
│    К  сожалению  MS-DOS  накладывает  более  серьезные  ограничения на имена
│ файлов  чем  iS-DOS.  В  связи  с  этим  возможны некоторые на первый взгляд
│ странные особенности работы плагина.
│    При копировании  каталога  содержащего  файлы  имена  которых отличаются
│ только  регистром  плагин  выдает  сообщение  о  существовании файла с таким
│ именем. Лечиться переименованием исходных файлов.
│    При копировании текстовых файлов из/в iS-DOS необходимо учитывать, то что
│ знаки переноса строки и конца файла в iS-DOS отличаются от принятых на PC.
